# 1강
## 노드의 정의
### 1. 노드의 정의
Node.js는 크롬V8 자바스크립트 엔진으로 빌드된 자바스크립트 런타임이다.
노드는 서버의 역활도 수행이 가능한 자바스크립트 런타임이며 노드로 자바스크립트로 작성된 서버를 실행할 수 있다.
서버 실행을 위해 필요한 http/https/http2 모듈제공 
### 2. 런타임
런타임: 특정 언어로 만든 프로그램들을 실행할 수 있게 해주는 가상 머신의 상태
### 3. 내부구조
노드는 V8과 libuv를 내부적으로 포함
V8엔진: 오픈 소스 자바스크립트 엔진 -> 속도 문제 개선
libuv:노드의 특성인 이벤트 기반,논블로킹(비동기) I/O모델을 구현한 라이브러리
## 노드의 특성
### 1. 이벤트 기반
이벤트가 발생할 떄 미리 지정해둔 작업을 수행하는 방식
* 이벤트의 예: 클릭, 네트워크 요청, 타이머등
* 이벤트 리스너: 이벤트를 등록하는 함수
* 콜백 함수: 이벤트가 발생했을 떄 실행될 함수 
#### 이벤트 발생 순서
이벤트 리스너에 콜백 함수 등록 -> 이벤트 발생 -> 등록된 콜백 함수 호출
### 2. 논블로킹 I/O
논 블로킹: 오래 걸리는 함수를 백그라운드로 보내서 다음 코드가 먼저 실행되게 하고, 나중에 오래 걸리는 함수를 실행
* 논블로킹 방식 하에서 일부 코드는 백그라운드에서 병렬로 실행된다.
* 일부코드: I/O작업(파일 시스템 접근, 네트워크 요청), 압축, 암호화 등
* 나머지 코드느 블로킹 방식으로 실행된다. I/O작업이 많을 떄 노드 활용성이 극대화된다.
### 3. 프로세스 vs 스레드
프로세스와 스레드
* 프로세스: 운영체제에서 할당하는 작업의 단위, 프로세스간 자원 공유X 예) 크롬창
* 스레드: 프로세스 내에서 실행되는 작업의 단위, 부모 프로세스 자원 공유 예) 크롬창의 탭들(네이버,유튜브등) 
노드 프로세스는 멀티 스레드이지만 직접 다룰 수 있는 스레드는 하나이기 때문에 싱글 스레드라고 표현
노드는 주로 멀티 스레드 대신 멀티 프로세스 활용
노드는 14버전부터 멀티 스레드 사용 가능
### 4. 싱글 스레드
싱글 스레드라 주어진 일을 하나밖에 처리하지 못한다.
- 블로킹이 발생하는 경우 나머지 작업은 모두 대기해야 한다. (비효율)
예) 식당의 점원이 한명(싱글 스레드)이 있다면 한명의 고객만 응답이 가능
### 5. 멀티 스레드 모델과의 비교
싱글 스레드는 에러를 처리하지 못하는 경우 멈춤
- 프로그래밍 난이도 쉽고, CPU, 메모리 자원 적게 사용
멀티 스레드는 에러 발생 시 새로운 스레드를 생성하여 동작
- 단, 새로운 스레드 생성이나 놀고 있는 스레드 처리에 비용 발생
- 프로그래밍 난이도 어려움
- 스레드 수만큼 자원을 많이 사용한다.
### 6. 멀티 스레드의 활용
노드 14버전
- 멀티 스레드를 사용할 수 있도록 worker_threads 모듈 도입
- CPU를 많이 사용하는 작업인 경우에 활용 가능
- 멀티 프로세실만 가능했던 아쉬움을 달래준다.

|멀티 스레딩|멀티 프로세싱|
|:---:|:---:|
|하나의 프로세스 안에서 여러 개의 스레드 사용|여러개의 프로세스 사용|
|CPU 작업이 많을 떄 사용|I/O요청이 많을 떄 사용|
|프로그래밍이 어렵다.|프로그레밍이 비교적 쉽다.|

## 노드의 역할
### 1. 서버로서의 노드
서버: 네트워크를 통해 클라이언트에 정보나 서비스를 제공하는 컴퓨터 또는 프로그램
클라이언트: 서버에 요청을 보내는 주체(브라우저, 데스크탑 프로그램, 모바일앱, 다른 서버에 요청을 보내는 서버)
- 브라우저(클라이언트, 요청)가 웹사이트(서버,응답)에 접속
- 핸드폰(클라이언트)을 통해 앱스토어(서버)에서 앱 다운로드
노드 != 서버 하지만 노드는 서버를 구성할 수 있게 하는 모듈을 제공한다.
### 2. 노드 서버의 장단점

|장점|단점|
|:---:|:---:|
|멀티 스레드 방식에 비해 컴퓨터 자원을 적게 사용|싱글 스레드라서 CPU코어를 하나만 사용|
|I/O작업이 많은 서버로 적합|CPU작업이 많은 서버로는 부적함|
|멀티 스레드 방식보다 쉽다.|하나뿐인 스레드가 먼추지 않도록 관리해야 한다.|
|웹 서버가 내장되어 있다.|서버 규모가 커졌을 떄 서버를 관리하기 어렵다.|
|자바스크립트를 사용한다.|어중간한 성능|
|JSON형식과 호환라기 쉽다.||

CPU작업을 위해 AWS Lambda나 Google Cloud Functions같은 별도 서비스 사용
페이팔, 넷플릭스, 나사, 월마트, 링크드인, 우버 등에서 메인 또는 서브 서버로 사용
### 3. 서버 외의 노드
자바스크립트 런타임이기 때문에 용도가 서버에만 한정되지 않음
웹, 모바일, 데스크탑 앱에도 사용
- 웹 프레임워크: Angular, React, Vue, Meteor 등
- 모바일 앱 프레임워크: React Native
- 데스크탑 개발도구: Electron(Atom, Slack, VSCode, Discord 등 제작)
